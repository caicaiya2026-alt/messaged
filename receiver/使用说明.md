# 消息接收端 - 独立运行版本

## 概述

这是一个独立的 Windows 可执行程序，无需安装 Python 即可运行。

## 系统要求

- **操作系统**: Windows 7 或更高版本
- **内存**: 至少 512 MB RAM
- **磁盘空间**: 至少 100 MB 可用空间
- **网络**: 支持局域网或互联网连接

## 文件说明

### 主要文件

- **消息接收端.exe** - 主程序（独立可执行文件）
- **config.json** - 配置文件（可选）

## 使用步骤

### 第一步：复制文件

1. 将 `消息接收端.exe` 复制到目标电脑
2. 可以复制到任何位置（桌面、文档等）

### 第二步：运行程序

双击 `消息接收端.exe` 启动程序

### 第三步：配置服务器

1. **主机地址**:
   - `0.0.0.0` - 监听所有网络接口（推荐）
   - `127.0.0.1` - 仅本机访问
   - `192.168.x.x` - 仅局域网访问

2. **端口**:
   - 默认: `8765`
   - 可修改为其他端口（1024-65535）

### 第四步：启动服务器

点击"启动服务器"按钮

### 第五步：连接发送端

在另一台电脑上打开发送端页面，输入服务器地址和端口。

## 配置选项

### 主机地址说明

| 地址 | 说明 | 适用场景 |
|------|------|---------|
| 0.0.0.0 | 监听所有网络接口 | 推荐用于局域网 |
| 127.0.0.1 | 仅本机访问 | 本地测试 |
| 192.168.x.x | 特定局域网IP | 限制特定网络 |

### 端口说明

- 默认端口: `8765`
- 有效范围: `1024 - 65535`
- 防火墙: 确保选择的端口未被防火墙阻止

## 连接方式

### 方式一：局域网连接

1. 接收端电脑（电脑B）：
   - 主机地址: `0.0.0.0`
   - 端口: `8765`
   - 点击"启动服务器"

2. 发送端电脑（电脑A）：
   - 打开发送端页面
   - 服务器地址: `192.168.x.x`（电脑B的IP）
   - 端口: `8765`
   - 点击"连接服务器"

### 方式二：内网穿透连接

1. 接收端电脑（电脑B）：
   - 主机地址: `0.0.0.0`
   - 端口: `8765`
   - 点击"启动服务器"
   - 启动 Cloudflare Tunnel（见 COMPLETE_SOLUTION.md）

2. 发送端电脑（电脑A）：
   - 打开发送端页面（Cloudflare Pages）
   - 服务器地址: `ws.your-domain.com`
   - 端口: `443`
   - 点击"连接服务器"

## 功能说明

### 主界面

- **服务器设置区域**: 配置主机地址和端口
- **连接状态**: 显示当前连接状态
- **消息显示区域**: 显示接收到的所有消息
- **清除按钮**: 清空消息历史
- **状态栏**: 显示服务器运行状态

### 消息显示

- 时间戳: 每条消息显示接收时间
- 来源地址: 显示发送方的IP地址
- 消息内容: 显示实际消息文本

## 常见问题

### Q: 双击 exe 文件没有反应？

A: 检查：
1. 是否被杀毒软件阻止
2. Windows Defender 是否拦截
3. 右键选择"以管理员身份运行"

### Q: 启动服务器失败？

A: 检查：
1. 端口是否被其他程序占用
2. 防火墙是否阻止连接
3. 尝试更换端口号

### Q: 发送端无法连接？

A: 检查：
1. 接收端是否正在运行
2. 服务器地址是否正确
3. 防火墙是否允许该端口
4. 两台电脑是否在同一网络（局域网）

### Q: 如何查看本机IP地址？

A: 在接收端电脑上：
```bash
ipconfig
```
查找 "IPv4 地址"。

### Q: 程序可以最小化到托盘吗？

A: 当前版本不支持，但可以：
1. 最小化窗口到任务栏
2. 程序会继续在后台运行

```json
{
  "server": {
    "host": "0.0.0.0",
    "port": 8765
  },
  "ui": {
    "window_width": 600,
    "window_height": 500
  }

```

### 防火墙配置

Windows 防火墙：
1. 打开"Windows Defender 防火墙"
2. 选择"允许应用通过防火墙"
3. 浏览并选择"消息接收端.exe"
4. 勾选"专用"和"公用"
5. 点击"确定"
